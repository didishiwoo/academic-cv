# 技术方案 — 个人简历主页重构

## 一、技术栈与框架现状

| 项目 | 内容 |
|------|------|
| 静态站点框架 | Hugo v0.161.1 |
| 主题框架 | Hugo Blox (Academic CV template) |
| 样式系统 | Tailwind CSS（通过 Hugo Blox 内置） |
| 部署方式 | GitHub Pages |
| 内容管理 | Markdown + YAML 数据文件 |
| 多语言 | 当前中英双语（Hugo 原生 i18n） |

## 二、架构决策

### 2.1 单页式布局方案
利用 Hugo Blox 已有的 `type: landing` 页面类型 + `block` 系统，将所有区块堆叠在一个页面内。不需要创建新的 Hugo 页面类型。

### 2.2 区块组成
按需求边界文档确定的 7 个区块，各区块使用以下 Hugo Blox 块类型：

| 区块 | 使用的块类型 | 数据来源 |
|------|-------------|---------|
| 1. 顶部身份区 | `resume-biography-3`（定制化） | `data/authors/me.yaml` + `content/zh/authors/me/_index.md` |
| 2. 教育背景 | `resume-experience` | `data/authors/me.yaml`（education 字段） |
| 3. 科研经历 | `markdown` 自定义块 | 直接写在 `_index.md` 的 text 字段 |
| 4. 荣誉奖项 | `resume-awards` | `data/authors/me.yaml`（awards 字段） |
| 5. 学术成果 | `collection`（精简） | `content/zh/publications/` 目录 |
| 6. 技能 | `resume-skills` | `data/authors/me.yaml`（skills 字段） |
| 7. 与导师研究的关联 | `markdown` 自定义块 | 直接写在 `_index.md` 的 text 字段（**核心新增内容**） |

### 2.3 语言方案
- 移除 `content/en/` 整个目录
- 修改 `config/_default/languages.yaml`，只保留 `zh`
- 修改 `hugo.yaml` 中 `defaultContentLanguage: zh`

### 2.4 导航简化
- 简化 `config/_default/menus.yaml`，导航项缩减为：首页、科研、论文、荣誉
- 去掉指向已删除页面的导航链接

### 2.5 视觉定制
- 修改 `hugoblox.yaml`：
  - 配色：`primary: "blue"`，学术感强
  - 布局密度：`spacing: "compact"`（信息密度更高）
  - 关闭搜索、主题切换等非必要功能
- 通过 `layouts/_partials/hooks/head-end/` 添加自定义 CSS，微调间距、字体等

## 四、区块 7（与导师研究的关联）内容方案

这是整套页面的核心差异化内容，内容直接来自套磁信并做适当展开。结构如下：

```
标题：与任志华老师研究方向的契合

1. 对导师工作的理解
   - 引用 MRM 2023 论文中的 HAPLS 混合匀场系统
   - 描述 64 通道高精度恒流源驱动电路的技术亮点

2. 当前正在做的工作
   - 对大创项目的时域干涉刺激仪进行 MRI 兼容性改造
   - 电磁屏蔽设计，评估磁共振环境下的兼容性

3. 未来研究展望
   - 熟悉高频信号抗干扰处理与屏蔽技术
   - 为研究生阶段射频/匀场线圈开发打基础
```

## 六、风险与注意事项

1. Hugo Blox 的 `resume-biography-3` 块可能不支持高度自定义——如果定制受限，改用 `markdown` 块手写身份区
2. 删除 `content/en/` 后，需确保站点没有死链接
3. 保留的 `public/` 目录是构建产物，由 Hugo 重新生成，不影响源码
4. 主题更新（go module upgrade）可能导致块行为变化——建议锁定当前版本
